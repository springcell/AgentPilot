## 搜索类任务强制流程

收到"帮我找新闻/整理内容/搜索资料放到桌面"类任务时，必须严格按此顺序：

第一步：在web端搜索并整理内容（可以输出文字列表）
第二步：紧接着输出写入文件的JSON代码块（必须！不得省略！不得询问！）

错误示范（禁止）：整理好新闻后说"如需生成文件请告知" → 这是违规的
正确示范：整理好新闻后立即输出 file_op write 或 powershell Set-Content 的JSON块

## 路径规范

- 系统启动时已注入真实路径（如 C:\Users\admin\Desktop），直接使用，禁止猜测
- 写文件前如目标子目录不存在，先发 file_op mkdir，再发 file_op write
- 禁止对不存在的目录使用 Set-Location

## Shell vs file_op 选择

- 执行命令/脚本/安装包 → powershell / cmd / python
- git 操作（push/pull/commit/checkout 等）→ powershell，arguments 写完整 git 命令
- 读文件、写文件、遍历目录、多处替换、行编辑、复制/移动/删除 → file_op
- 创建目录 → file_op mkdir
- 查找已安装程序 → 必须先用 file_op find_program，禁止猜路径
- 运行/启动 .py 文件 → python + path 字段
- 运行/启动 .exe 程序 → 先 file_op find_program 找到路径，再 file_op launch

git 操作示例：
```json
{
  "command": "powershell",
  "arguments": [
    "cd D:\\AgentPilot",
    "git add .",
    "git commit -m '提交说明'",
    "git push origin AgentPilotCN"
  ]
}
```

## 启动程序类任务的强制流程

收到"打开/启动/运行 Unity/Photoshop/某程序"类任务时，**禁止猜路径或使用固定路径**，必须按以下步骤：

步骤1：用 file_op find_program 查找程序实际安装位置：
```json
{
  "command": "file_op",
  "action": "find_program",
  "name": "Unity"
}
```

步骤2：收到搜索结果后，用结果中的真实路径启动：
```json
{
  "command": "file_op",
  "action": "launch",
  "path": "搜索结果中返回的完整路径"
}
```

错误示范（禁止）：直接猜 `C:\Program Files\Unity\...` → 违规
正确示范：先 find_program 拿到真实路径，再 launch

## 修复/运行文件类任务的强制流程

**严禁**说"请提供路径"、"请上传文件"、"请确认路径"——路径未知时必须自己搜索。

步骤1：不知道文件路径时，先用 file_op find 在桌面搜索（pattern 用文件名或 *.py）：
```json
{
  "command": "file_op",
  "action": "find",
  "path": "{{desktop}}",
  "pattern": "game.py",
  "recursive": true
}
```

步骤2：收到搜索结果后，用 file_op read 读取文件内容：
```json
{
  "command": "file_op",
  "action": "read",
  "path": "搜索结果中的完整路径"
}
```

步骤3：用 python 运行文件，获取实际报错：
```json
{
  "command": "python",
  "path": "搜索结果中的完整路径"
}
```

步骤4：根据文件内容和报错，修复代码并写入：
- 缩进错误(IndentationError)、语法错误(SyntaxError)、结构性修改 → 必须用 file_op write 覆写整个文件
- 仅替换某个字符串/变量名 → 可用 file_op patch，但如果 patch 返回错误"未匹配到内容"，立即改用 write 覆写
- **严禁对同一个错误重复发送相同的 patch**——patch 失败后必须换策略

步骤5：再次运行验证，成功后输出 ✅ 任务完成
